AMIGA_drawChunky8

AMIGA_drawChunky8 sets graphics rendering on 16 bits Chunky instead of Bitplanes

new AMIGA_drawChunky8(_chunkyBuffer: any, _srcW: any, _srcH: any, _dstW: any, _dstH: any)
Parameters
_chunkyBuffer (any) : Source buffer. UInt8Array containing the pixels values
_srcW (any) : width in pix of the source buffer (so _srcW * 2 bytes per line)
_srcH (any) : height in pix of the source buffer
_dstW (any) : width in pix of the destination buffer (screen)
_dstH (any) : height in pix of the destination buffer (screen)

▸ __READ_ME_FIRST__()

JS_ASM_Tools is a series of helpers to navigate smoothly between "regular" js and asm. You can invoke all its methods through singleton "TOOLS", using TOOLS.methodname(args). For example: regs.d[0]=TOOLS.MULS_68K(79,regs.d[0]);

__READ_ME_FIRST__()
▸ MULS_68K(s, d)

Emulates the MULS asm instruction: s.w * d.w ==> (s*d).l.

MULS_68K(s: number, d: number): number
Parameters
s (number) source (16 bit signed int)
d (number) destination (16 bit signed int). NOT MODIFIED AS A M68K WOULD DO: Result is returned, not copied to arg2.
Returns
number: s.w * d.w (javascript number)
Throws
  • any: error (alert box) if params>32767 or <-32768
Example
let yOfs = TOOLS.MULS_68K(40, lineY);
regs.d[0] = TOOLS.JSInt32ToAsm(TOOLS.MULS_68K(79, TOOLS.toInt16(regs.d[0]))); // remember that regiters are stored as unsigned long while JS variables are signed integers, so need to convert
▸ DIVS_68K(_source, _dest)

Partially emulates the DIVS asm instruction: _dest.l / _source.w.

DIVS_68K(_source: number, _dest: number): number
Parameters
_source (number) source (16 bit signed int)
_dest (number) destination (32 bit signed int). NOT MODIFIED AS A M68K WOULD DO: Result is returned, not copied to arg2.
Returns
number: _dest.l/_source.w (javascript number). DOES NOT RETURN REMAINDER IN THE MSB LIKE THE M68K WOULD DO
Throws
  • any: error (alert box) if _source>32767 or _source<-32768
  • any: error (alert box) if _source == 0
  • any: error (alert box) if division overflow
Example
let slope = TOOLS.DIVS_68K(delta, iter);
regs.d[0] = TOOLS.JSInt32ToAsm(TOOLS.DIVS_68K(TOOLS.toInt32(regs.d[0]), TOOLS.toInt16(regs.d[1]))); // remember that regiters are stored as unsigned long while JS variables are signed integers, so need to convert
